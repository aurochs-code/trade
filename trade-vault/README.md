# trade-vault

这是交易系统的人读知识库。MySQL 是事实源，`trade-vault` 只保留规则、当前状态、每日巡检结论和复盘，不再保存每日原始日志、信号快照、当日输出等机械生成文件。

目录约定：

- `00-系统`：系统使用、交易规则、Agent 协作约束
- `01-状态`：账户、持仓、池子等当前状态摘要
- `02-巡检`：每日收盘后由 Agent 生成的系统和交易巡检报告
- `03-分析`：周复盘、月复盘、专题分析、历史留档
- `04-决策`：今日决策、候选池和需要人工关注的交易结论

使用约定：

- Agent 只能通过仓库根目录的 `bin/trade ...` 或 `bin/trade mcp` 获取事实，不直接执行内部 `.py` 文件。
- 每日收盘后生成一份 `02-巡检/YYYY-MM-DD.md`，汇总当天 pipeline、事件、订单、风控、人工确认和异常。
- 历史逐日原始输出不再保存在 vault；需要追溯事实时查询 MySQL 事件流和运行记录。
- Obsidian 可以直接打开本目录作为 vault 根。
