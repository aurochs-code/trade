# 系统使用

所有操作统一通过仓库根目录的 `bin/trade` 调用。运行时数据库是 MySQL，必须配置 `ASTOCK_DATABASE_URL`。Agent 不直接执行 `src/astock_trading/**/*.py`。

## 常用命令

```bash
# 系统健康
bin/trade doctor --json
bin/trade agent-context --json

# 数据库
bin/trade db status --json
bin/trade db tables --json
bin/trade db check --json

# 运行记录和事件
bin/trade runs list --json
bin/trade events query --json

# 持仓和人工成交补录
bin/trade status --json
bin/trade record-buy CODE SHARES PRICE --yes --json
bin/trade record-sell CODE SHARES PRICE --yes --json

# 选股和候选池
bin/trade screener run --query "近20日放量突破，市值100亿以上" --json
bin/trade screener score --codes 002138,300558 --json
bin/trade screener candidates --json
bin/trade screener promote 002138 --to core --json
bin/trade screener reject 002138 --reason "不符合当前风格" --json

# 人工确认交易
bin/trade manual-trades list --json
bin/trade manual-trades confirm REQUEST_ID --json
bin/trade manual-trades reject REQUEST_ID --json

# 模拟盘
bin/trade paper status --json
bin/trade paper orders --json

# MCP
bin/trade mcp
```

## Pipeline

Pipeline 通过 `bin/trade run-pipeline NAME --json` 执行。盘中轮巡可以多次运行，日级任务需要保持幂等。

| pipeline | 目的 |
|---|---|
| morning | 盘前状态、市场信号、候选池和今日决策 |
| intraday_monitor | 盘中持仓和风控轮巡 |
| noon | 午间检查 |
| evening | 收盘状态、风控和运行总结 |
| scoring | 核心池评分 |
| weekly | 周复盘素材 |

## 每日巡检

收盘后 Agent 生成 `trade-vault/02-巡检/YYYY-MM-DD.md`。巡检报告只写结论，不复制全部原始事件。

必须覆盖：

- 当天 pipeline 成功/失败情况
- 事件流中的交易建议、订单、成交、取消和风控事件
- 人工确认交易的待确认、已确认、已拒绝状态
- 模拟盘订单和持仓变化
- 数据源、DB、MCP、CLI 异常
- 明天开盘前需要人工处理的事项

## MCP

MCP 是 Agent 的标准工具入口，使用同一套 MySQL 数据和 application services。需要查看可用工具时运行：

```bash
bin/trade agent-context --json
```
